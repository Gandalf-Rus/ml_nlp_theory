# План проекта: Система анализа эмоций в тексте

## 1. Общее описание задачи

Многоуровневая классификация текста:

- **Уровень 1:** Тональность — Позитивная / Негативная / Нейтральная _(мультикласс, 1 метка)_
- **Уровень 2:** Подвиды эмоций — конкретизация эмоциональной окраски текста из 9 эмоций$^*$ _(мультилейбл, несколько меток)_
- **Уровень 3:** Стиль — Официальный / Неофициальный _(бинарная классификация)_
- **Языки:** Английский → Русский → Сербский

$^*$ 9 эмоций:
* sadness (грусть)
* anger (злость) 
* fear (страх)
* disgust (отвращение/неприязнь)
* anticipation (предвкушение)
* joy (радость)
* surprise (удивление)
* gratitude (благодарность)
* love (любовь)

```
['sadness', 'anger', 'fear', 'disgust', 'anticipation', 'joy', 'surprise', 'gratitude', 'love']
```

### Этап 1 — Данные и [[EDA]]

**Сбор и выбор датасетов:**

| Датасет                  | Назначение                                | Источник                                        |
| ------------------------ | ----------------------------------------- | ----------------------------------------------- |
| GoEmotions (Google)      | Мультилейбл эмоции, 27 классов + нейтраль | HuggingFace                                     |
| SST-2 / IMDB             | Позитив/негатив, бейзлайн сентимента      | HuggingFace / Kaggle                            |
| SemEval 2018 Task 1      | Интенсивность эмоций                      | CodaLab                                         |
| ISEAR                    | 7 базовых эмоций                          | Kaggle                                          |
| Formality Style Stenford | Официальный / неофициальный стиль         | HuggingFace (osyvokon/pavlick-formality-scores) |
|                          |                                           |                                                 |

*Доп информация о выбранных датасетах:*

* **GoEmotions (от Google):** Главный датасет для **Модели B**$^*$ (подвиды эмоций). Содержит 58 000 комментариев с Reddit. Размечен на 27 эмоций, что требует маппинга в наши 8 целевых классов. Это датасет формата _мультилейбл_ (один текст может иметь несколько меток).

* **SST-2 или IMDB:** Датасеты для **Модели A**$^*$ (тональность). IMDB — это 50 000 отзывов на фильмы, четко поделенных на позитивные и негативные. SST-2 — похожий датасет, но состоящий из более коротких предложений (его отбросили). Мы добавим к ним тексты с нейтральной меткой из GoEmotions, чтобы получить три класса (Позитив / Негатив / Нейтраль).

* **GYAFC (Grammarly’s Yahoo Answers Formality Corpus):** Датасет для **Модели C**$^*$ (стиль текста). Содержит параллельные предложения: одно написано неформальным сленгом, другое — переписано в строгом официальном стиле. Идеально для бинарной классификации.

$^*$ про эти модели подробней написано в блоке [[#Этап 4 — Базовые модели (Baseline)]]


**Работа с данными:**
> *Подробное [[Описание процесса работы с данными (подготовки)]]*
- Маппинг чужих меток на нашу схему (грусть, злость, страх, отвращение/неприязнь, предвкушение, радость, удивление, благодарность)
- Разметка официальный/неофициальный стиль (GYAFC + эвристики)
- Разбивка: Train 70% / Validation 15% / Test 15%


**Разведочный анализ данных (EDA):**
> *Подробное [[Описание процесса разведочного анализа данных (EDA)]]*

- Распределение классов и дисбаланс
- Анализ длины текстов
- Частотный анализ слов (топ-слова по каждой эмоции)
- Визуализация: wordcloud, гистограммы, heatmap корреляций меток

## 4. Математическая база

| Тема                                                        | Зачем нужна                                                                                     |
| ----------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| Линейная алгебра (векторы, матрицы, скалярное произведение) | [[Эмбеддинги]], операции с весами, [[Геометрия нейросетевых эмбеддингов и агрегация признаков]] |
| Теорема Байеса, условная вероятность                        | Naive Bayes                                                                                     |
| Логистическая функция, Softmax и Sigmoid                    | Классификация, выходной слой                                                                    |
| Производные, градиенты, Chain Rule                          | Backpropagation                                                                                 |
| Функции потерь (Cross-Entropy, BCE, Hamming Loss)           | Обучение моделей                                                                                |
| Градиентный спуск (SGD, Adam)                               | Оптимизация нейросетей                                                                          |
| TF-IDF (статистика слов)                                    | Классическая векторизация                                                                       |
| Косинусное сходство                                         | Сравнение эмбеддингов                                                                           |
| Механизм Attention / Self-Attention                         | Трансформеры, BERT                                                                              |
| Regularization (L1, L2, Dropout)                            | Борьба с переобучением                                                                          |
| Функции активации (ReLU, Tanh)                              | Нелинейность в скрытых слоях нейросетей (CNN, RNN)                                              |
| Информационная энтропия (Entropy, Gini)                     | Логика работы деревьев решений (Random Forest, Бустинг)                                         |
| Произведение Адамара ($\odot$)                              | Механизмы фильтрации (gates) в архитектурах LSTM и GRU                                          |
| Агрегация метрик (Micro/Macro/Weighted)                     | Точный расчет метрик для мультилейбл-задач (подвиды эмоций)                                     |

## 6. Порядок изучения алгоритмов

```
BoW/TF-IDF + Logistic Regression / Naive Bayes / SVM
                    ↓
          RNN → LSTM / GRU → Bi-LSTM
                    ↓
         BERT fine-tuning (английский)
                    ↓
       XLM-RoBERTa (русский + сербский)
```

Каждый шаг объясняет, почему следующий лучше — в этом и есть главная учебная ценность проекта.
