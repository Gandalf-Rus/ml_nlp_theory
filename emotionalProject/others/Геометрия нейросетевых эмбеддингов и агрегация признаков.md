# Конспект: Геометрия нейросетевых эмбеддингов и агрегация признаков

>[!WARNING]
> Word2Vec/FastText не на строгой L2-нормализации

Для успешной работы с Моделью B (эмоции), Моделью A (тональность) и Моделью C (формальность) на Этапе 5 необходимо понимать механику перехода от разреженных пространств к плотным тензорным структурам.

## 1. Структура данных: Тензоры в NLP

Алгоритмы машинного обучения оперируют массивами чисел, которые называются тензорами. Форма тензора определяется количеством его осей (измерений).

* **Подход TF-IDF (Этап 3-4):**
  Каждый текст описывался вектором-строкой фиксированной длины (10000 словарных токенов). Вся обучающая выборка представляла собой строгий двумерный тензор (матрицу) размерности $(N, 10000)$, где $N$ — количество текстов. Алгоритм Linear SVC способен напрямую принимать такую структуру.

* **Подход Word2Vec/FastText (Этап 5):**
  Алгоритм преобразует не текст целиком, а **каждое отдельное слово** в плотный вектор (ненормализированый) фиксированной длины $D$ (в нашем случае $D=300$). 
  Если текст состоит из $T$ слов, он математически представляется как двумерный тензор (матрица) размерности $(T, 300)$.
  Поскольку тексты имеют разную длину, $T$ — величина переменная. Набор из $N$ текстов формирует неравномерную структуру, которую нельзя напрямую подать в классические алгоритмы, требующие жесткой размерности матрицы признаков $(N, D)$.

## 2. Геометрия семантического пространства

Вектор слова в Word2Vec/FastText $v_t \in \mathbb{R}^{300}$ — это координата точки в 300-мерном пространстве.
* Оси этого пространства не имеют интерпретируемых названий (в отличие от TF-IDF, где каждая ось — конкретное слово). Это скрытые семантические признаки, выведенные алгоритмом при факторизации матриц совместной встречаемости.
* Косинусное расстояние между двумя точками (векторами слов) отражает их семантическую близость. Синонимы образуют плотные кластеры (облака точек).

## 3. Агрегация признаков: Mean Pooling

Чтобы решить проблему переменной длины $T$ и передать текст в алгоритм Linear SVC, необходимо сжать матрицу $(T, 300)$ в единый вектор $(300,)$. Оптимальный метод для базовых архитектур — усреднение (Mean Pooling).

**Математическая логика:**
Выполняется покоординатное суммирование векторов всех $T$ слов текста с последующим скалярным делением на их количество:
$$V_{doc} = \frac{1}{T} \sum_{t=1}^{T} v_t$$
Где:
* $V_{doc}$ — результирующий вектор документа размерности $(300,)$.
* $v_t$ — вектор $t$-го слова в документе.
* $T$ — общее количество слов в документе, векторы которых присутствуют в словаре модели.

**Геометрическая логика (Центроид):**
Представьте 300-мерное пространство, в котором каждое из $T$ слов текста является точкой. Эти точки формируют геометрическое облако. 
Операция вычисляет **центроид** — точный геометрический центр масс данного облака точек. 

**Пример механики:**
Если текст состоит из набора слов, векторы которых сгруппированы в области пространства, отвечающей за "радость" (например, *joy, happy, wonderful*), их усредненный вектор (центроид) также будет находиться в этой области. При добавлении слова из противоположного кластера (например, *terrible*), центроид математически сместится по соответствующим осям, отражая изменение общего семантического вектора (направления) всего документа.

Полученные векторы $V_{doc}$ для всех $N$ текстов формируют итоговую матрицу признаков размерности $(N, 300)$, которая подается на вход Linear SVC для вычисления оптимальной разделяющей гиперплоскости.

> Эта операция **агрегации** (сжатия размерности). В архитектурах машинного обучения она называется **Mean Pooling**.
