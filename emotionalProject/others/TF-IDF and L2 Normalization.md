# TF-IDF (Term Frequency - Inverse Document Frequency)

>[!INFO]
>- статистика слов 
>- **Для чего нужно:** Классическая векторизация. Превращение текста в числа с учетом важности слов.
>- **Пример:** Понять математически, что уникальное слово «разочарован» для классификации важнее, чем часто встречающийся предлог «в».


Это основа для классической векторизации. 
Алгоритмы не умеют читать буквы, им нужны цифры. 
Самый простой способ превратить текст в цифры — **Bag of Words (BoW)**: просто посчитать, сколько раз каждое слово встретилось в тексте. 
Но у BoW есть огромный минус: токены вроде «я», «это», «на» будут иметь самые большие числа, хотя смысла для определения эмоций в них ноль.

**TF-IDF** решает эту проблему элегантным математическим трюком: он штрафует токены, которые встречаются слишком часто во всех текстах, и поощряет редкие, но смысловые токены.

Формула состоит из двух частей:

- **TF (Term Frequency — частота слова):** Считает, насколько часто токен встречается в _конкретном_ предложении.
    
    $$TF = \frac{\text{Количество появлений токена в тексте}}{\text{Общее количество токенов в этом тексте}}$$
    
    _Например: Если в отзыве из 10 слов токен «ужасно» встречается 2 раза, то $TF = 0.2$._
    
- **IDF (Inverse Document Frequency — обратная частота документа):**
    
    Оценивает, насколько это слово уникально в масштабах _всего твоего датасета_.
    
    $$IDF = \log\left(\frac{\text{Общее число всех текстов датасета}}{\text{Число текстов, где встречается этот токен}}\right)$$
    
    _Например: токен «и» есть в 9999 текстах из 10000. $IDF = \log(10000 / 9999) \approx 0$. А токен «ужасно» есть только в 50 текстах. $IDF = \log(10000 / 50) \approx 2.3$.
    
* **Сглаживание IDF (Smooth IDF)**

По умолчанию `scikit-learn` использует сглаживание (`smooth_idf=True`), чтобы избежать математической ошибки деления на ноль (если признак вдруг имеет нулевую частоту). В числитель и знаменатель добавляется константа 1, а к итоговому результату логарифмирования прибавляется еще 1, чтобы полностью исключить обнуление весов.

$$IDF = \log\left(\frac{N + 1}{N_t + 1}\right) + 1$$

_(Где $N$ — общее число текстов, $N_t$ — число текстов с данным токеном)_
    

**Итог:** Мы умножаем эти значения ($TF \times IDF$). Для токена «и» итоговый вес будет умножен на ноль (оно игнорируется). А вес токена «ужасно» сильно вырастет. Модель получит вектор, где самые большие числа стоят у действительно важных маркеров эмоций.

Для каждого слова мы проводим подобный подсчёт. Рассчитывая его потенциальную ценность.
```
Токен => номер сллва из колонки + его вес в этом предложении
```

Вот как это выглядит структурно:
 * Колонки (Columns): Это наш фиксированный словарь из 10 000 слов, собранный на этапе fit. Каждому слову намертво привязан свой порядковый номер. Слово bad — это колонка №45. Слово good — колонка №112.
 * Строки (Rows): Это наши тексты (конкретные отзывы). Строка №1 — это первый отзыв.
 * Ячейки (Cells): На пересечении строки №1 и колонки №45 записывается только одно число — тот самый итоговый вес TF * IDF.
 
Это число в ячейке отвечает на один конкретный вопрос: насколько сильно слово bad характеризует именно этот отзыв на фоне всех остальных десятков тысяч отзывов?
Если в отзыве этого слова нет вообще, в ячейке будет стоять 0.0.


## Команды у TF-IDF

* **`fit`** (обучиться) - алгоритм читает данные и составляет словарь из $n$ хаданных топ слов и вычисляет, насколько каждое слово важно (тот самый вес IDF).
* **`transform`** (применить) - беруться текста и просто пропускаем их через _уже готовый_ словарь, собранный с помощью `fit`.

## Более дотошный порядк действий

На этапе `.fit()` (только на `train`):
1. Алгоритм собирает топ-10000 слов и фиксирует: `Слово => i-ый номер колонки`.
2. Алгоритм считает **IDF** (Inverse Document Frequency) для каждого из 10000 слов. Он запоминает теоретическую редкость слова на основе всего корпуса `train`. На этом работа `.fit` закончена. Он ничего не умножает.
    
На этапе `.transform()` (на `train` и на `test`):
1. Алгоритм берет конкретный текст.
2. Считает **TF** (Term Frequency) — частоту слова _только внутри этого одного текста_.
3. Берет заготовленный ранее IDF для этого слова, умножает `TF * IDF` и записывает итоговый результат в ячейку таблицы.
4. Применяет L2-нормализацию ко всей строке вычисленных значений $TF \times IDF$, приводя евклидову длину итогового вектора текста к 1.0.


# L2 Normalization

>[!INFO]
> L2-нормализация выравнивает длины всех векторов.
> Приведение всех векторов к единичным векторам для дальнейшего косинусного сравнения 

**Как это работает математически:** 
Алгоритм берет строку таблицы, берет все его значения tf-idf столбцов, возводит каждое в квадрат, складывает, извлекает корень (это называется вычислением Евклидовой длины вектора). А затем **делит каждое число в строке на эту общую длину**.

**Что это дает геометрически:** 
Длинный текст имеет огромные значения весов. Короткий текст имеет маленькие значения. После L2-нормализации вектор длинного текста "сжимается", а вектор короткого "растягивается" так, чтобы их итоговая геометрическая длина стала строго равна **1.0**.



$$V_{norm}​ = \frac{V}{\sqrt{x^2 + y^2 + ...}}$$
