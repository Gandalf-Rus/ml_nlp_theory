
Для того чтобы Трансформер мог начать вычисления, текст должен пройти через три стадии трансформации. На этом этапе решаются проблемы неизвестных слов, разной длины предложений и формализуется вход для библиотек (HuggingFace Transformers).

#### 1. Алгоритм токенизации WordPiece (Проблема неизвестных слов)

В классическом NLP словари собирались из целых слов. Если размер словаря 30 000 слов, а в тексте встретилась опечатка или неологизм (например, "unbelievableee"), модель получала ошибку OOV (Out-Of-Vocabulary) и заменяла слово на бесполезный токен `[UNK]` (Unknown).

**Решение BERT (WordPiece):**

BERT использует токенизацию на уровне **сабвордов** (подслов). Алгоритм WordPiece имеет фиксированный словарь (для английского BERT это 30 522 токена), который состоит из самых частых целых слов, слогов и отдельных букв.

**Как это работает алгебраически:**

1. Если слово популярное (например, `"bank"`), оно токенизируется целиком $\rightarrow$ `["bank"]`.
    
2. Если слово редкое или неизвестное (например, `"unbelievable"`), алгоритм жадно дробит его на известные словарю куски $\rightarrow$ `["un", "##believ", "##able"]`.
    
3. **Смысл префикса `##`:** Две решетки означают, что этот токен является _продолжением_ предыдущего куска, а не началом нового слова.
    
4. **Математический итог:** Благодаря этому словарю из 30 522 элементов, модель математически способна представить **абсолютно любое слово** или опечатку во вселенной, просто разбив его вплоть до одиночных символов. Проблема OOV полностью уничтожена. Сеть может уловить смысл приставки "un-" даже у слова, которое видит впервые в жизни.
    

#### 2. Батчинг, Padding и Attention Mask (Защита матрицы вычислений)

**Проблема размерностей при обучении на GPU:**

Видеокарты (GPU) вычисляют данные матрицами. Мы не можем подать в модель одно предложение из 10 слов, а второе из 50. Нам нужно собрать "батч" (например, 32 текста) в единый прямоугольный тензор строгой формы $32 \times L$.

**Шаг 1: Padding (Выравнивание)**

Алгоритм находит самое длинное предложение в батче (например, 50 токенов). Все остальные предложения искусственно удлиняются до 50 с помощью добавления пустого токена `[PAD]` (в словаре BERT он имеет индекс `0`).

_Текст:_ `[CLS] I love this [SEP] [PAD] [PAD] [PAD]...`

**Шаг 2: Attention Mask (Предотвращение отравления паддингом)**

Если мы просто пропустим эту матрицу через механизм Self-Attention, произойдет математическая катастрофа. Векторы реальных слов своими запросами ($Q$) начнут перемножаться с ключами ($K$) токенов `[PAD]`. Вектор токена `[CLS]` впитает в себя смысловой мусор из пустых ячеек.

Чтобы это предотвратить, параллельно с текстом генерируется матрица **Attention Mask**.

- Это бинарный массив той же длины $L$, где `1` означает реальное слово, а `0` — токен `[PAD]`.
    
- **Как маска работает внутри Attention:** До применения функции `softmax` в формуле $\frac{Q \cdot K^T}{\sqrt{d_k}}$, алгоритм проверяет маску. В те ячейки матрицы сходства, где маска равна `0`, алгоритм принудительно вписывает минус бесконечность ($-\infty$).
    
- **Итог:** Поскольку $e^{-\infty} = 0$, функция `softmax` выдает для токенов `[PAD]` строго $0\%$. Механизм внимания математически «слепнет» к пустотам. `[PAD]` токены присутствуют в тензоре ради сохранения формы для GPU, но абсолютно не участвуют в логике нейросети.
    

#### 3. Требования библиотеки Transformers (Формат входа)

При написании кода на PyTorch + HuggingFace, предобученный BERT ожидает получить на вход не сырые строки, а строго структурированный словарь (Dictionary), состоящий из трех тензоров:

1. **`input_ids`:** Тензор с целочисленными индексами токенов.
    
    _(Пример: `[101, 1045, 2293, 102, 0, 0]`)_ — где `101` это всегда `[CLS]`, `102` это `[SEP]`, а `0` это `[PAD]`. Именно этот тензор внутри модели превратится в матрицу эмбеддингов $X$.
    
2. **`attention_mask`:** Бинарный тензор для блокировки паддинга.
    
    _(Пример: `[1, 1, 1, 1, 0, 0]`)_.
    
3. **`token_type_ids` (Segment IDs):** Бинарный тензор, указывающий принадлежность слова к определенному предложению. В Этапе 7 (задача NSP) мы передавали два предложения. Для первого предложения этот тензор заполнялся нулями (`0`), для второго — единицами (`1`).
    
    _Важный нюанс для проекта:_ Поскольку решается задача классификации одного текста (пост/комментарий), не нужно второе предложение. Поэтому этот тензор всегда будет состоять **только из нулей** _(Пример: `[0, 0, 0, 0, 0, 0]`)_.
    

#### Главный вывод Этапа 9

Перед попаданием в Трансформер сырой текст проходит строгую математическую стандартизацию. Использование сабвордов (WordPiece) позволяет ограниченному словарю понимать бесконечный язык и справляться с опечатками. Добавление паддинга решает физическую проблему вычислений на GPU (прямоугольные тензоры), а применение Attention Mask защищает внутреннюю логику Attention от информационного отравления пустыми токенами. Итогом этой стадии являются три целочисленных тензора (`input_ids`, `attention_mask`, `token_type_ids`), которые становятся стартовым базисом для работы 12 слоев BERT.
