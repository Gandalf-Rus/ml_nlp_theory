### Расшифровка метрик

Чтобы понимать метрики, нужно знать четыре базовых термина из матрицы ошибок (Confusion Matrix) для каждого класса:

- **True Positive (TP):** Модель предсказала класс A, и это действительно класс A.
    
- **False Positive (FP):** Модель предсказала класс A, но на самом деле это другой класс.
    
- **False Negative (FN):** Это был класс A, но модель предсказала другой класс.
    
- **True Negative (TN):** Это не класс A, и модель сказала, что это не класс A.
    

**Precision (Точность):**

Показывает долю правильных ответов среди всех предсказаний _данного класса_.

$$Precision = \frac{TP}{TP + FP}$$

**Recall (Полнота):**

Показывает долю найденных объектов _от всех объектов этого класса_ в датасете.

$$Recall = \frac{TP}{TP + FN}$$

**F1-score:**

Это гармоническое среднее между Precision и Recall. Оно наказывает модель за сильный дисбаланс между этими двумя метриками.

$$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$

Метрика равна 1.0 при идеальном предсказании и 0.0 при полном провале.

**Support (Поддержка):**

Это физическое количество текстов данного класса в тестовой выборке `x_test`.

**Accuracy (Общая точность):**

Доля абсолютно верных предсказаний по всему датасету.

$$Accuracy = \frac{\text{Все } TP}{\text{Все тексты}}$$


### Конкретизация к каждой метрике

Когда в датасете классы распределены неравномерно (например, в Модели B у нас 942 текста с радостью и всего 131 со страхом), мы не можем просто сказать "модель работает на 80%". Нам нужно понимать, _как именно_ усреднялись результаты между такими разными по размеру группами.

Вот разбор каждого метода усреднения из отчета `classification_report`:

#### 1. Macro avg (Макро-усреднение)

- **Как считает:** Вычисляет метрику (например, F1) для каждого класса по отдельности, а затем находит их обычное среднее арифметическое. Размеры классов полностью игнорируются.
    
- **Формула:** $\frac{F1_{joy} + F1_{fear} + \dots + F1_{n}}{N}$
    
- **Смысл:** **Все классы равны.** Голос крошечного класса "страх" (131 пример) имеет точно такой же вес, как голос гигантского класса "радость" (942 примера).
    
- **Зачем нужен:** Это самая жесткая метрика. Если алгоритм отлично находит популярные классы, но полностью "ослеп" к редким, `macro avg` сильно упадет. Мы смотрели именно на него, чтобы понять, справился ли SVM с редкими эмоциями.
    

#### 2. Weighted avg (Взвешенное усреднение)

- **Как считает:** Вычисляет метрику для каждого класса, но при усреднении умножает её на долю этого класса в датасете (на его `support`).
    
- **Формула:** $(F1_{joy} \cdot \text{Доля}_{joy}) + (F1_{fear} \cdot \text{Доля}_{fear}) + \dots$
    
- **Смысл:** **Все тексты равны.** Крупные классы перетягивают итоговую оценку на себя.
    
- **Зачем нужен:** Показывает, как модель будет вести себя "в среднем по больнице" на реальном потоке данных. Если модель завалит редкий класс, `weighted avg` почти не изменится, потому что тексты этого класса попадаются редко.
    

#### 3. Micro avg (Микро-усреднение)

- **Как считает:** Вообще стирает границы между классами. Он сваливает все предсказания в одну глобальную корзину. Он считает суммарное количество всех верных ответов (True Positives), ложных срабатываний (False Positives) и пропусков (False Negatives) по всему датасету разом, и из этих глобальных цифр выводит метрику.
    
- **Смысл:** Оценивает каждое индивидуальное решение "да/нет" как независимое событие, независимо от того, к какому классу оно относилось.
    
- **Зачем нужен:** В многоклассовой задаче (Модель А) `micro avg` всегда математически равен общей `accuracy` (точности). В задачах с мульти-лейблами (Модель B) он показывает глобальную пропорцию правильно угаданных ярлыков ко всем возможным ярлыкам.
    

#### 4. Samples avg (Усреднение по примерам)

- **Как считает:** Этот показатель появляется **только в задачах Multi-label** (как наша Модель B с эмоциями). Он считает метрику не по столбцам (классам), а по строкам (текстам).
    
- **Смысл:** Оценивает, насколько алгоритм угадал _полный набор_ эмоций для каждого конкретного текста. Например, если в тексте были `[страх, удивление]`, а алгоритм выдал только `[страх]`, он получает за этот текст 0.5. Затем эти оценки усредняются по всем 4073 текстам.
    
- **Зачем нужен:** Показывает, насколько хорошо модель справляется с "пакетом" ярлыков для одного пользователя/текста за один раз.
    

---

**Краткая шпаргалка для анализа:**

1. Хочешь узнать, не игнорирует ли модель редкие классы? Смотри на **Macro avg**.
2. Хочешь узнать общую эффективность на реальном потоке данных? Смотри на **Weighted avg**.
3. Решаешь задачу, где у одного текста может быть сразу несколько ярлыков? Смотри на **Samples avg**.
4. `Micro avg` показывает, какова вероятность того, что система примет правильное решение для _любого случайного документа_ из общего потока. `Weighted avg` показывает, насколько хорошо модель работает _в среднем по классам_, делая поправку на то, что одни классы встречаются в реальной жизни чаще других.
